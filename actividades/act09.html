Red Team Report: Pentesting My File Server 1

Evaluación de Seguridad Ofensiva - Actividad 09
Riesgo Crítico

Executive Summary

Se realizó una auditoría de seguridad integral sobre el servidor de archivos corporativo "My File Server 1". La evaluación identificó deficiencias críticas en la configuración de servicios y obsolescencia en el kernel del sistema operativo, permitiendo un compromiso total del sistema, desde el acceso inicial anónimo hasta la obtención de privilegios de superusuario (root).

Alcance y Metodología

Objetivo Técnico

Evaluar la resistencia del servidor ante ataques externos y movimientos laterales, identificando vectores de entrada y fallos de configuración.

Metodología PTES

Se aplicó el estándar Penetration Testing Execution Standard, cubriendo fases de reconocimiento, enumeración, explotación y post-explotación.

Walkthrough del Pentesting

1

Fase de Reconocimiento

Identificación de servicios activos mediante escaneo de puertos profundo:

nmap -sV -sC -p- 192.168.1.XX

Puertos Identificados: 21 (FTP), 22 (SSH), 80 (HTTP), 139/445 (SMB).

2

Enumeración de Servicios

Se detectó acceso FTP Anónimo habilitado, permitiendo la descarga de archivos sensibles del servidor:

  • Archivo credenciales.txt: Contenía listas de posibles usuarios corporativos.
  • Documentos PDF con diagramas de red interna.

Simultáneamente, la enumeración de SMB reveló recursos compartidos sin restricciones de lectura para invitados.

3

Fase de Explotación

Utilizando la información obtenida en la fase anterior, se realizó un ataque de fuerza bruta dirigido sobre el servicio SSH, logrando acceso inicial con el usuario identificado.

ssh usuario@192.168.1.XX
4

Post-Explotación: Escalada de Privilegios

Tras obtener una shell limitada, se analizó la versión del kernel del sistema, identificando vulnerabilidad al exploit Dirty COW (CVE-2016-5195).

uname -a # Kernel 2.6.22 < 3.9

Se compiló y ejecutó el exploit localmente, permitiendo sobrescribir archivos de solo lectura y obteniendo una shell con privilegios de root.

Análisis de Impacto (Modelo CIA)

Pilar Impacto Observado
Confidencialidad Alto. Acceso total a archivos privados y credenciales de usuarios.
Integridad Alto. Capacidad de modificar archivos de sistema y configuraciones críticas de red.
Disponibilidad Medio. Posibilidad de interrumpir servicios, aunque no fue el objetivo principal del ejercicio.

Roadmap de Remediación

Fase 1: Contención

Deshabilitar el acceso FTP anónimo y rotar todas las contraseñas comprometidas de forma inmediata.

Fase 2: Fortalecimiento

Actualizar el kernel del sistema operativo para mitigar vulnerabilidades de escalada de privilegios y endurecer configuraciones de SMB/SSH.

Fase 3: Monitoreo

Implementar sistemas de detección de intrusos (IDS) y auditorías de integridad de archivos periódicas.
