Introducción

Esta actividad documenta la creación de una VPN IPSec Site-to-Site en Packet Tracer utilizando routers Cisco 1941 con licencias Security K9. El objetivo principal es establecer un túnel cifrado que garantice la confidencialidad e integridad de los datos entre dos sedes remotas conectadas a través de un ISP. Para ello, se configuran políticas ISAKMP de Fase 1 y conjuntos de transformación de Fase 2 para proteger el tráfico "interesante" de la red.

Diseño y Topología de Red

La infraestructura consta de tres routers (R1, R3 e ISP), dos switches 2960 y terminales finales en dos subredes distintas.

Topología: R1 (192.168.1.1) ↔ ISP ↔ R3 (192.168.3.1)

1. Activación de Licencias de Seguridad

Es obligatorio habilitar el paquete de seguridad en los routers Cisco 1900 para disponer de los comandos de criptografía.

CISCO IOS
Router# configure terminal
Router(config)# license boot module c1900 technology-package securityk9
Router(config)# exit
Router# reload

2. Configuración de VPN Fase 1 (ISAKMP)

Se define la política de gestión de claves con cifrado AES de 256 bits y autenticación por clave compartida.

R1 CONFIG
R1(config)# crypto isakmp policy 10
R1(config-isakmp)# encryption aes 256
R1(config-isakmp)# authentication pre-share
R1(config-isakmp)# group 5
R1(config-isakmp)# exit
R1(config)# crypto isakmp key secretkey address 209.165.200.1

3. Configuración de VPN Fase 2 (IPSec)

Se establece el Transform Set y el Crypto Map para definir cómo se cifrarán los paquetes de datos.

R1 CONFIG
R1(config)# crypto ipsec transform-set R1-R3 esp-aes 256 esp-sha-hmac
R1(config)# crypto map IPSEC-MAP 10 ipsec-isakmp
R1(config-crypto-map)# set peer 209.165.200.1
R1(config-crypto-map)# set transform-set R1-R3
R1(config-crypto-map)# match address 100
R1(config-crypto-map)# exit
R1(config)# interface gigabitEthernet 0/0
R1(config-if)# crypto map IPSEC-MAP
Tráfico Interesante: Se configura la lista de acceso (ACL 100) para permitir el tráfico entre las subredes 192.168.1.0/24 y 192.168.3.0/24.

4. Pruebas de Conectividad y Verificación

Tras realizar el ping inicial, se verifica que el túnel esté activo y cifrando paquetes.

VERIFICACIÓN
R1# show crypto isakmp sa
R1# show crypto ipsec sa

Conclusiones

La implementación fue exitosa, validada por el estado QM_IDLE en la fase de gestión de claves y el cifrado activo de paquetes confirmado mediante el comando show crypto ipsec sa. Este ejercicio refuerza la importancia de los protocolos de seguridad para mitigar riesgos de interceptación en infraestructuras de red corporativas.

Topología: Red Site-to-Site configurada con Routers Cisco 1941 y túnel IPSec activo.
