Lab 01: SQL injection vulnerability in WHERE clause allowing retrieval of hidden data

Apprentice

Indicaciones

Esta aplicación tiene una vulnerabilidad de inyección SQL en el filtro de categorías de productos. Cuando el usuario selecciona una categoría, la aplicación realiza una consulta SQL para recuperar detalles de los productos publicados de dicha categoría. El objetivo es inyectar un ataque que haga que la aplicación muestre todos los productos de todas las categorías, tanto publicados como no publicados.

Procedimiento Técnico

  • Utilizar Burp Suite para interceptar la petición cuando se selecciona un filtro de categoría (por ejemplo, 'Gifts').
  • Localizar el parámetro category en la petición GET.
  • Modificar el valor del parámetro agregando la carga maliciosa para romper la lógica de la consulta.
' OR 1=1--

Verificación y Explicación

Al insertar el payload, la consulta resultante en el servidor se transforma de una búsqueda filtrada a una condición universalmente verdadera. La comilla simple ' cierra la cadena de la categoría, OR 1=1 genera una sentencia verdadera para cada fila de la tabla, y los guiones -- comentan el resto de la consulta original (como el filtro de productos no publicados). Como resultado, la base de datos devuelve la totalidad de los registros de productos al servidor.
