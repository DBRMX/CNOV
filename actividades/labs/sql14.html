Lab 14: Blind SQL injection with time delays

Practitioner

Indicaciones

Este laboratorio contiene una vulnerabilidad de inyección SQL en la cookie de seguimiento (TrackingId). La aplicación procesa la consulta SQL de forma asíncrona, por lo que no hay cambios en la respuesta HTTP ni errores visibles cuando la consulta falla. El objetivo es confirmar la vulnerabilidad provocando un retardo de 10 segundos en la respuesta del servidor.

Procedimiento Técnico

  • Interceptar la petición que contiene la cookie TrackingId mediante Burp Suite.
  • Identificar el motor de base de datos (PostgreSQL en este escenario) para utilizar la función de retardo correcta.
  • Modificar el valor de la cookie inyectando una sentencia que concatene el ID original con un comando de suspensión del sistema.
  • Enviar la petición y verificar en la pestaña "Timeline" de Burp Suite o en el navegador que el servidor tarda exactamente 10 segundos adicionales en responder.
' ; SELECT pg_sleep(10)--

Verificación y Explicación

La inyección basada en tiempo es una variante de "Blind SQLi" utilizada cuando el servidor no ofrece ninguna pista visual sobre el éxito de una consulta. Al inyectar comandos como pg_sleep(), obligamos al motor de base de datos a pausar su ejecución antes de devolver la respuesta. Si el tiempo de carga de la página aumenta proporcionalmente al valor inyectado, se confirma que el atacante tiene control sobre la ejecución de comandos SQL en el servidor, permitiendo incluso ataques más complejos de extracción de datos.
