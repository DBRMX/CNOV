VOLVER A HALL OF FAME

Lab 10: SQL injection UNION attack, listing the database contents on Oracle

Practitioner

Indicaciones

Este laboratorio presenta una vulnerabilidad de inyección SQL en el filtro de categorías de productos. El objetivo es recuperar los nombres de las tablas y columnas en una base de datos **Oracle** para obtener las credenciales del administrador. El laboratorio se resuelve al iniciar sesión con la cuenta de administrator.

Procedimiento Técnico

  • Determinar el número de columnas (en este caso son 2) utilizando UNION SELECT NULL, NULL FROM dual--.
  • Identificar la tabla de usuarios consultando la vista de sistema all_tables.
  • Una vez localizada la tabla (ej. USERS_ABCDEF), listar sus columnas consultando all_tab_columns filtrando por el nombre de la tabla encontrada.
  • Extraer los nombres de usuario y contraseñas de las columnas identificadas para obtener la clave del administrador.
' UNION SELECT table_name, NULL FROM all_tables--

Verificación y Explicación

En Oracle, el acceso a los metadatos se realiza a través de vistas como all_tables y all_tab_columns, en lugar del estándar information_schema. Además, es mandatorio que cada consulta SELECT tenga una cláusula FROM válida; por ello, durante la fase de reconocimiento estructural se emplea la tabla dual. El éxito del ataque radica en navegar correctamente por estas tablas de catálogo para reconstruir el esquema privado de la aplicación y localizar las tablas de autenticación que suelen tener nombres generados dinámicamente.
