VOLVER A HALL OF FAME

Lab 08: SQL injection UNION attack, retrieving the database type and version on MySQL and Microsoft

Practitioner

Indicaciones

La aplicación presenta una vulnerabilidad de inyección SQL en el filtro de categorías. El objetivo es ejecutar un ataque UNION que permita recuperar la versión de la base de datos. El laboratorio se considera resuelto cuando la página renderiza la información de la versión de **MySQL** o **Microsoft SQL Server**.

Procedimiento Técnico

  • Determinar el número de columnas devueltas por la consulta original (en este caso son 2).
  • Identificar qué columna es compatible con tipos de datos string para visualizar la versión.
  • Inyectar el payload utilizando la variable global @@version y cerrar la consulta con el carácter de comentario específico para este motor.
' UNION SELECT @@version, NULL%23

Verificación y Explicación

A diferencia de Oracle, motores como MySQL y Microsoft SQL Server no requieren la cláusula FROM para consultas de valores constantes o variables de sistema. La variable @@version contiene la información detallada del motor en ejecución. Un detalle crítico es el uso de # (codificado como %23 en la URL) para comentar el resto de la consulta, ya que el uso de guiones -- requiere un espacio posterior que a veces el navegador puede truncar.
