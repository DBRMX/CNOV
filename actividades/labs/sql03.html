Lab 03: SQL injection UNION attack, determining the number of columns returned by the query

Practitioner

Indicaciones

Este laboratorio contiene una vulnerabilidad de inyección SQL en el filtro de categorías de productos. El objetivo es determinar el número de columnas devueltas por la consulta original utilizando un ataque UNION. El laboratorio se resolverá exitosamente cuando se inyecte un payload que determine la cantidad exacta de columnas sin provocar un error en la aplicación.

Procedimiento Técnico

  • Interceptar con Burp Suite la petición del filtro de categoría.
  • Utilizar la técnica de valores NULL para evitar errores de tipo de datos mientras se busca la estructura.
  • Incrementar el número de valores NULL en la sentencia UNION SELECT hasta que la página cargue correctamente sin devolver un error HTTP 500.
' UNION SELECT NULL,NULL,NULL--

Verificación y Explicación

Para que un ataque UNION sea exitoso, la consulta inyectada debe devolver el mismo número de columnas que la consulta original. Al inyectar una serie de NULL, estamos probando la estructura de la tabla de forma segura. En este caso específico, al inyectar tres valores NULL, la aplicación dejó de mostrar un error de servidor, lo que confirma que la consulta original recupera exactamente tres columnas de la base de datos.
