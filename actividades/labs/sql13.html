Lab 13: Visible error-based SQL injection

Practitioner

Indicaciones

Este laboratorio contiene una vulnerabilidad de inyección SQL en la cookie de seguimiento (TrackingId). La aplicación muestra mensajes de error detallados de la base de datos en la respuesta HTTP. El objetivo es utilizar estos mensajes de error para extraer la contraseña del usuario administrator de la tabla users e iniciar sesión.

Procedimiento Técnico

  • Identificar que la aplicación devuelve errores detallados al inyectar caracteres especiales en la cookie TrackingId.
  • Provocar un error intencionado de conversión de tipos de datos utilizando la función CAST().
  • Realizar una subconsulta para obtener la contraseña del administrador y forzar su conversión a un tipo de dato incompatible (como un entero).
  • Leer la contraseña que aparece directamente dentro del mensaje de error del servidor renderizado en la página.
' AND 1=CAST((SELECT password FROM users LIMIT 1) AS int)--

Verificación y Explicación

La inyección basada en errores visibles aprovecha una configuración insegura del servidor que expone detalles técnicos internos. Al intentar convertir una cadena de texto (la contraseña exfiltrada) en un número entero, el motor de base de datos genera una excepción. Debido a que la aplicación está configurada para mostrar errores de depuración, el valor de la contraseña se incluye en el texto del mensaje de error, permitiendo una extracción de datos mucho más rápida que en los ataques ciegos tradicionales.
