Lab 17: Blind SQL injection with out-of-band data exfiltration

Practitioner

Indicaciones

Este laboratorio utiliza una cookie de seguimiento (TrackingId) vulnerable a inyección SQL. El objetivo final es exfiltrar la contraseña del usuario administrator. Debido a que la aplicación es totalmente "ciega" a las técnicas convencionales, se debe forzar a la base de datos a realizar una petición externa que incluya los datos sensibles dentro del nombre de dominio consultado.

Procedimiento Técnico

  • Configurar el cliente de Burp Collaborator para obtener un dominio de escucha.
  • Utilizar una subconsulta SQL para recuperar la contraseña de la tabla users.
  • Construir un payload de red (Oracle EXTRACTVALUE) que concatene la contraseña recuperada como un prefijo del subdominio del Collaborator.
  • Enviar el payload en la cookie de seguimiento y revisar la pestaña de Collaborator para interceptar la consulta DNS.
  • Extraer el valor de la contraseña directamente de la petición entrante (por ejemplo, contraseña.ID-COLLABORATOR.oastify.com).

Verificación y Explicación

La exfiltración de datos fuera de banda (OOB) es una de las técnicas más potentes en el arsenal de un auditor de seguridad. Al combinar la vulnerabilidad de inyección con la capacidad del servidor para resolver nombres de dominio o realizar peticiones HTTP, el atacante puede extraer información sin depender de la respuesta del servidor web original. En este caso, la base de datos actúa como un cliente que, al intentar resolver el nombre de dominio malicioso, nos entrega la contraseña "de contrabando" dentro de la propia trama de la consulta DNS.
