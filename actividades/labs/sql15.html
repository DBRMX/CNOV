Lab 15: Blind SQL injection with time delays and information retrieval

Practitioner

Indicaciones

Este laboratorio contiene una vulnerabilidad de inyección SQL ciega en la cookie de seguimiento (TrackingId). La aplicación procesa la consulta de forma asíncrona y no devuelve ninguna diferencia en la respuesta según el éxito de la consulta. El objetivo es utilizar retardos de tiempo condicionales para recuperar la contraseña del usuario administrator.

Procedimiento Técnico

  • Confirmar que el motor de base de datos responde a retardos de tiempo (PostgreSQL) inyectando una pausa de 10 segundos.
  • Utilizar una sentencia CASE dentro de la inyección para ejecutar pg_sleep() únicamente cuando una condición sea verdadera.
  • Determinar la longitud de la contraseña del administrador observando los tiempos de respuesta del servidor.
  • Configurar Burp Intruder para iterar sobre cada posición de la contraseña y cada carácter posible, identificando la letra correcta cuando la respuesta tarda 10 segundos adicionales.
' ; SELECT CASE WHEN (username='administrator' AND SUBSTR(password,1,1)='a') THEN pg_sleep(10) ELSE pg_sleep(0) END FROM users--

Verificación y Explicación

La exfiltración de datos mediante tiempo es la técnica más avanzada de Blind SQLi cuando no existen canales de respuesta visual o de error. Al condicionar la función de sueño (pg_sleep) al éxito de una subconsulta, convertimos el tiempo de respuesta del servidor en un canal binario de información. Si el servidor tarda en responder, sabemos que el carácter probado es correcto; si responde de inmediato, es incorrecto. Este método permite reconstruir cadenas de texto completas de forma remota y silenciosa.
