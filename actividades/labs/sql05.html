VOLVER A HALL OF FAME

Lab 05: SQL injection UNION attack, retrieving data from other tables

Practitioner

Indicaciones

Este laboratorio contiene una vulnerabilidad de inyección SQL en el filtro de categorías de productos. La aplicación utiliza los resultados de la consulta SQL para mostrarlos en la respuesta. El objetivo es utilizar un ataque UNION para recuperar todos los nombres de usuario y contraseñas de la tabla users. El laboratorio se resuelve al iniciar sesión como el usuario administrator utilizando las credenciales extraídas.

Procedimiento Técnico

  • Determinar el número de columnas devueltas por la consulta original (en este caso son 2).
  • Verificar qué columnas son compatibles con el tipo de datos de cadena de texto (ambas son compatibles).
  • Inyectar la sentencia UNION SELECT para solicitar los campos username y password de la tabla users.
  • Localizar las credenciales del usuario administrator en los resultados renderizados en la página y utilizarlas para acceder.
' UNION SELECT username, password FROM users--

Verificación y Explicación

El operador UNION permite combinar el conjunto de resultados de dos consultas separadas en una sola respuesta. Al conocer la estructura de la base de datos (nombres de tabla y columnas), podemos forzar a la aplicación a "unir" los productos legítimos con los datos sensibles de la tabla de usuarios. Dado que la aplicación está diseñada para listar strings en la interfaz, las contraseñas exfiltradas se vuelven visibles directamente para el atacante.
