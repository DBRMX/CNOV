VOLVER A HALL OF FAME

Lab 09: SQL injection UNION attack, listing the database contents on non-Oracle databases

Practitioner

Indicaciones

Este laboratorio contiene una vulnerabilidad de inyección SQL en el filtro de categorías de productos. El objetivo es extraer de la base de datos los nombres de todas las tablas y sus columnas para localizar la tabla de usuarios y exfiltrar las credenciales del administrador. El laboratorio se resuelve al iniciar sesión como administrator.

Procedimiento Técnico

  • Determinar el número de columnas y cuáles aceptan texto (2 columnas detectadas).
  • Listar las tablas de la base de datos consultando information_schema.tables.
  • Identificar la tabla de usuarios (ej. users_abcdef) y listar sus columnas consultando information_schema.columns.
  • Extraer los datos de las columnas identificadas (ej. username_xyz y password_xyz) de la tabla de usuarios.
' UNION SELECT table_name, NULL FROM information_schema.tables--

Verificación y Explicación

En la mayoría de los sistemas de gestión de bases de datos relacionales (RDBMS) excepto Oracle, existe un esquema estándar llamado information_schema que actúa como un diccionario de datos. Al realizar un ataque UNION contra las vistas tables y columns de este esquema, un atacante puede reconstruir visualmente la estructura de la base de datos sin previo conocimiento de la misma. Esto permite identificar tablas con nombres aleatorios o personalizados que contienen información crítica de autenticación.
