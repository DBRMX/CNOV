Lab 07: SQL injection UNION attack, retrieving the database type and version on Oracle

Practitioner

Indicaciones

Este laboratorio contiene una vulnerabilidad de inyección SQL en el filtro de categorías de productos. El objetivo es utilizar un ataque UNION para recuperar la versión de la base de datos. El laboratorio se resuelve cuando la página muestra el banner de la versión de **Oracle**.

Procedimiento Técnico

  • Determinar el número de columnas (en este caso son 2) utilizando UNION SELECT NULL, NULL FROM dual--. Es importante notar que en Oracle, toda sentencia SELECT debe incluir la cláusula FROM.
  • Verificar qué columnas aceptan texto inyectando valores en lugar de NULL.
  • Consultar la tabla de sistema v$version para extraer el banner informativo de la base de datos.
' UNION SELECT BANNER, NULL FROM v$version--

Verificación y Explicación

Oracle Database tiene una sintaxis estricta que requiere que cada consulta apunte a una tabla. Para pruebas estructurales se utiliza la tabla integrada dual. Una vez establecida la compatibilidad de columnas, accedemos a v$version, una vista de rendimiento dinámico que contiene el nombre y la versión del software de la base de datos. Esta información es vital en un pentest para identificar vulnerabilidades específicas asociadas a versiones antiguas del motor.
