Lab 11: Blind SQL injection with conditional responses

Practitioner

Indicaciones

Este laboratorio utiliza una cookie de seguimiento (TrackingId) para recopilar datos analíticos. La consulta SQL que procesa esta cookie no devuelve resultados directamente en la página, pero la aplicación muestra el mensaje "Welcome back" solo si la consulta devuelve registros. El objetivo es realizar una inyección a ciegas para obtener la contraseña del usuario administrator.

Procedimiento Técnico

  • Verificar la vulnerabilidad inyectando una condición verdadera (' AND 1=1--) y una falsa (' AND 1=2--) en la cookie, observando la presencia/ausencia del mensaje "Welcome back".
  • Confirmar la existencia de la tabla users y del usuario administrator mediante subconsultas lógicas.
  • Determinar la longitud de la contraseña probando condiciones como ' AND (SELECT 'a' FROM users WHERE username='administrator' AND LENGTH(password)>19)='a'.
  • Exfiltrar la contraseña carácter por carácter utilizando la función SUBSTR() combinada con un ataque de fuerza bruta en Burp Intruder.
' AND (SELECT SUBSTR(password,1,1) FROM users WHERE username='administrator')='a

Verificación y Explicación

La inyección SQL a ciegas se basa en realizar preguntas de "sí o no" a la base de datos. Al manipular la cookie con una condición AND, forzamos a que la consulta original solo sea exitosa si nuestra condición inyectada también lo es. Si la letra probada es correcta, el servidor devuelve el mensaje de bienvenida; de lo contrario, el mensaje desaparece. Este proceso iterativo permite reconstruir información compleja, como credenciales, sin que estas se impriman nunca directamente en la interfaz.
