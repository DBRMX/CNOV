Lab 06: SQL injection UNION attack, retrieving multiple values in a single column

Practitioner

Indicaciones

Este laboratorio presenta una vulnerabilidad de inyección SQL en el filtro de categorías. El objetivo es realizar un ataque UNION que permita recuperar todos los nombres de usuario y contraseñas de la tabla users. A diferencia de otros laboratorios, aquí la consulta original solo devuelve una columna de texto útil, por lo que es necesario concatenar los datos para visualizarlos simultáneamente. El laboratorio se resuelve al iniciar sesión como administrator.

Procedimiento Técnico

  • Determinar que la consulta original devuelve dos columnas, pero solo una permite renderizar texto en la página.
  • Utilizar el operador de concatenación adecuado para el motor de base de datos (en este caso || con un separador como '~').
  • Inyectar el payload en la columna compatible con strings para obtener una lista de credenciales con el formato usuario~contraseña.
  • Identificar y copiar la contraseña del administrador para completar el reto.
' UNION SELECT NULL, username||'~'||password FROM users--

Verificación y Explicación

La concatenación es una técnica esencial cuando el número de columnas que se muestran en la interfaz web es limitado. Al unir el nombre de usuario y la contraseña con un carácter especial intermedio, podemos exfiltrar ambos datos en un solo campo de texto. Este método permite optimizar el ataque UNION y superar restricciones de diseño en la visualización de los resultados por parte de la aplicación.
