Entorno de Amenazas

Es la composición de software malicioso (malware), manipulación psicológica (ingeniería social) y explotación de vulnerabilidades técnicas[cite: 635].

Malware

Software diseñado para interrumpir operaciones, secuestrar datos (Ransomware) o minar criptomonedas (Cryptojacking)[cite: 637, 642].

Ingeniería Social

Manipulación para robar credenciales (Pretexting) o acceso físico siguiendo a empleados (Tailgating)[cite: 645, 646].

Ataques de Red

Saturación de servidores mediante Botnets (DDoS) o interceptación de tráfico (Man-in-the-Middle)[cite: 648, 649].

Vulnerabilidades de Sistema

Hardware

  • Rowhammer: Corrupción de datos mediante interferencias eléctricas en RAM[cite: 654].
  • Meltdown & Spectre: Rompen el aislamiento entre aplicaciones y el Kernel de la CPU[cite: 655, 656].

Software

  • Buffer Overflow: Desbordamiento de memoria para inyectar código malicioso[cite: 658].
  • Firmware Malicioso: Actualizaciones legales infectadas con malware (ej. NotPetya)[cite: 660].

El Eslabón más Débil

La mayoría de los incidentes no ocurren por fallos técnicos complejos, sino por errores, mal uso e ingeniería social del usuario[cite: 668, 670].

  • Brechas por cadenas de suministro: 30%[cite: 671].
  • Principales vectores: Credenciales robadas y Phishing[cite: 675, 679].

Estrategias de Mitigación

Autenticación Robusta: Uso de caracteres complejos para mitigar ataques de fuerza bruta y pulverización[cite: 687].
Cifrado de Comunicaciones: Implementación de WPA3 para proteger datos bancarios de sniffers[cite: 688].
Concientización Continua: Educar para evitar el uso de USBs externos y el parcheo constante de sistemas[cite: 691, 692].
