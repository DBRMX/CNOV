Spear Phishing Simulation: Social Engineer Toolkit

Auditoría de Ingeniería Social - Actividad 15
Riesgo Alto

Executive Summary

Este documento detalla la ejecución de una simulación de ataque de ingeniería social realizada en un entorno controlado. El objetivo principal fue evaluar la efectividad de las defensas perimetrales y la capacidad de respuesta del usuario ante técnicas de suplantación de identidad (Phishing). Se logró la infiltración exitosa mediante un servidor SMTP configurado localmente y el despliegue de una página de captura de credenciales.

Infraestructura Técnica

Servidor de Correo (MTA)

Uso de Postfix instalado en Linux para la gestión y retransmisión de paquetes SMTP.

Módulo de Explotación

Social Engineer Toolkit (SET) configurado para ejecutar campañas de Mass Mailer con plantillas HTML.

Canal de Salida

Relay configurado con Google SMTP (puerto 587) utilizando App Passwords para evadir bloqueos de seguridad.

Walkthrough del Ataque

1

Configuración del Relay SMTP

Se modificó el archivo /etc/postfix/main.cf para activar la autenticación SASL y el uso de TLS.

relayhost = [smtp.gmail.com]:587
smtp_sasl_auth_enable = yes
smtp_use_tls = yes
2

Gestión de Credenciales en SET

Dentro de SET, se seleccionó el vector Mass Mailer Attack y se optó por crear una plantilla personalizada (One-Time Use Email Template). Se configuró un asunto persuasivo para incentivar el clic del usuario.

3

Ejecución y Resultados

Tras la inyección del correo en el servidor de retransmisión, el sistema notificó el envío exitoso. Al hacer clic en el enlace, el usuario fue dirigido a una página de inicio de sesión falsificada diseñada para recolectar sus datos institucionales.

Análisis de Riesgos (Modelo CIA)

Confidencialidad (Alta)

Compromiso total de cuentas institucionales, permitiendo el acceso no autorizado a información privada y bases de datos.

Integridad (Media)

Posibilidad de realizar ataques Man-in-the-Middle o alterar registros financieros tras el acceso inicial.

Vulnerabilidad Crítica: La ausencia de registros SPF, DKIM y DMARC permitió que el servidor Postfix suplantara identidades sin ser bloqueado por filtros de origen.
