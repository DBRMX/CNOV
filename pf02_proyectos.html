PR02 - Parte 1: Investigación comparativa de plataformas de simulación de phishing

Fecha: 06/Mar/2026

Indicaciones

El objetivo de este proyecto es evaluar las capacidades del factor humano como sensor activo de amenazas mediante el análisis de soluciones tecnológicas líderes. Se establecieron los siguientes requerimientos:

  • Identificar y analizar las métricas críticas para la evaluación del comportamiento humano ante ataques de ingeniería social.
  • Realizar un estudio comparativo de las 8 plataformas más relevantes del mercado (KnowBe4, Proofpoint, Cofense, entre otras).
  • Determinar los diferenciadores estratégicos y la orientación técnica de cada solución analizada.
  • Establecer los fundamentos éticos y estratégicos necesarios para la implementación de un programa de concientización profesional.

Explicación Técnica

La investigación se fundamenta en la transición de un modelo de cumplimiento normativo hacia uno de resiliencia operativa. El análisis técnico destaca tres pilares de medición fundamentales:

  1. Tasa de Apertura y Clic (Click Rate): Métrica de vulnerabilidad inicial que identifica la exposición del usuario.
  2. Tasa de Reporte (Reporting Rate): Indicador de éxito más crítico que mide la proactividad del empleado al alertar sobre amenazas.
  3. Puntuación de Riesgo (Risk Score): Cálculo predictivo que cruza el historial de comportamiento, el departamento y el nivel de acceso a datos sensibles.

Finalmente, se concluye que la efectividad de un programa de seguridad humana depende de la integración de principios como el "Teachable Moment" (educación inmediata tras el error) y la privacidad por diseño.

PR02 - Parte 2: Simulador interactivo de Phishing (Desarrollo)

Fecha: 19/Mar/2026

Indicaciones

Implementación de un sistema interactivo para evaluar el reconocimiento de amenazas de ingeniería social. Los requerimientos incluyeron:

  • Desarrollo de 10 escenarios realistas con vectores de ataque variados (urgencia, autoridad, dominios engañosos).
  • Integración de un aviso ético y consentimiento informado antes de iniciar la prueba.
  • Sistema de retroalimentación inmediata que explique los indicadores de Phishing tras cada respuesta.

Explicación Técnica

El simulador utiliza lógica de programación para gestionar el flujo de preguntas, cronometrar el tiempo de respuesta por escenario y calcular el porcentaje de precisión final. Los datos de desempeño se almacenan localmente para asegurar la persistencia entre sesiones y permitir el análisis comparativo posterior.

PR02 - Parte 3: Scoreboard y Análisis estadístico de vulnerabilidad

Fecha: 19/Mar/2026

Indicaciones

Creación de una plataforma de visualización de datos para monitorear la resiliencia organizacional. Los objetivos clave fueron:

  • Diseñar un Ranking Global dinámico basado en puntuación y eficiencia de tiempo.
  • Implementar tarjetas de métricas que muestren el acierto promedio y el nivel de participación.
  • Identificar automáticamente el escenario más complejo basándose en el tiempo máximo observado.

Explicación Técnica

Se implementaron algoritmos de ordenamiento para procesar los objetos de datos del ranking. El análisis estadístico extrae el valor máximo de los tiempos de respuesta para determinar cuál vector de ataque es más efectivo en engañar al sensor humano. Este módulo permite transformar datos crudos en inteligencia accionable para programas de concientización futuros.
